---
title: 工作空间变量
---

> 类似于人类协作的方式，可能存在 **多个角色** 共同编辑同一个变量。

所以定义了一个用于公共操作的空间“**工作空间变量**”，在提示词模板中使用“**space**”作为标记。


**工作空间变量** 分为以下三种类型，以满足不同的数据处理需求：


```js tab="覆盖型变量"
- 此类变量在赋值时将覆盖原有数据，所以只会保留最后一次的值。

例如：space.name 格式为 String
space.name = "张三"
space.name = "李四"

变量存储的值为：
李四

```
```js tab="追加型变量"

- 追加型变量存储的是一个列表，每次赋值都会将新数据追加到原有变量之后，形成数据列表，适用于需要收集多个数据的场景。

例如：space.name 格式为 Array<String>
space.name = "张三"
space.name = "张三"

变量存储的值为：
['张三','张三']

```

```js tab="不重复追加型变量"

- 追加型变量的变种，在追加数据的同时，系统将自动执行排重处理，确保重复的数据不会被多次追加，防止有重复数据。
例如：space.name 格式为 Array<String>
space.name = "张三"
space.name = "张三"
space.name = "李四"

变量存储的值为：
['张三','李四']

张三  只保留第一次的数据
```
