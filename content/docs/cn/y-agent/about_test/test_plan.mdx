## 测试计划

测试计划的作用是已经确定的测试功能，统一加入一个计划，方便测试流程标准化。
测试计划的制定需要考虑以下几个方面：

- 测试目标：测试目标是什么，测试的目的和意义是什么。
- 选择合适的测试用例：测试用例要有代表性，并且全面。

### 测试计划的分类
测试计划页面的左上角，可以添加以下分类：
- 流程测试：针对流程进行测试，和 流程测试 菜单内功能相似。
- 单元测试：针对模型进行测试，和 单元测试 菜单内功能相似。

### 创建测试计划
按照需要创建相应类型的测试计划。

创建好测试计划后，需要为测试计划添加相应的测试案例。

可以通过两种方式添加测试案例：
- 关联用例库：可以直接选取用例库中的分类，直接添加到计划。
- 关联用例：可以只添加个别的用例到计划。

两种方式可以混合使用。

注意：测试计划只是一个对用例的引用，并不会创建用例的副本，用例修改后，测试计划内关联的用例也会同步修改。

### 编辑测试计划
同上述。

具体每一项的填写，根据测试计划的类型不同，请参考： [流程测试](/docs/y-agent/about_test/add_test#流程测试的配置参数). [单元测试](/docs/y-agent/about_test/add_test#单元测试的配置参数).


### 执行测试计划

测试计划配置好后，可以点击 右侧 执行 按钮，执行计划。

> 执行测试计划的时候，请认真填写 测试报告名称 ，这样可以在测试报告列表中，准确分辨出每次测试的目的。

计划是在后台执行的，所以无需保持登录状态。

注意：测试计划的执行是常耗时操作，请在系统空闲时进行。同时，请勿重复执行计划。计划在执行中，不要停止后台服务，否则会造成计划的中断。

测试计划执行完成后，可以通过 查看报告 按钮，或者 在测试报告菜单中查看执行结果，测试计划历次执行的结果，会按顺序保存在 查看报告 菜单中。

同时，计划关联的所有用例里面，也可以看到 运行结果。
