## 背景

由于在电商场景中，商品名称在销售端、产品资料中、仓储系统中、订单系统中、物流系统等各个系统中都有不同名称，

这会导致查询资料的时候无法准确匹配，并且大模型也无法理解商品之间的关系。

所以，在电商场景中，商品名称的唯一性是非常重要的。因此，商品管理功能，就是为了解决这一问题。
## 功能介绍

第一步应该建立商品管理档案。
然后，在店铺商品中，建立SKU和商品档案之间的关系。
### 商品档案管理

商品档案管理功能，是为了将不同系统中使用的商品名称统一管理起来。

可以帮助电商客户管理商品的名称，并保证商品名称的唯一性。

| 字段名     | 说明                                                                 |
| :--------- | :------------------------------------------------------------------- |
| 商品编码   | 外部系统唯一标识，可以理解为商家ERP系统中的商品编码。                  |
| 商品名称   | 本系统使用的商品的真实名称，应该准确命名，避免歧义。                   |
| 商品条码   | 选填，建议按照实际69码填写。                                           |
| 商品类目   | 帮助管理人员管理商品使用，对于本系统运行，无实际意义。                 |

### 店铺商品管理

需要新建店铺，然后在店铺商品中，建立SKU和商品档案之间的关系。

一下是新增商品档案的属性介绍：
| 字段名     | 说明                                                         |
| :--------- | :----------------------------------------------------------- |
| 商品SKU    | 商品的唯一标识，可以理解为商家ERP系统中的商品编码。            |
| 商品档案   | 商品的名称、条码、类目等信息。                                 |
| 店铺       | 选择店铺，商品归属哪个店铺。                                   |
| 销售状态   | 预留字段，暂未使用。                                           |
| 推荐标记   | 预留字段，暂未使用。                                           |
| 商品卖点   | 预留字段，暂未使用。                                           |
| 套装标记   | 此处会自动填写。                                               |

如何添加店铺商品：
在关联档案中，根据右侧商品编码、条码，搜索 商品档案 中已经添加的商品。

点击“添加”按钮，即可添加商品。

如果，一个商品SKU，对应多个商品档案，可以多次添加。此时，自动升级成套装。

如果，对应的数量不一致，可以手动调整配置数量。例如：商品档案中只有单支装，而店铺中某个SKU对应双支装，可以将配置数量填 2。
